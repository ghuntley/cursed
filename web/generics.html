Generics

Generics are so fetch right now! They let you write code that's giving main character energy with any data type. It's like having one function template that works with the whole squad.

Generic Stack Implementation
fr fr Define a generic Stack type
be_like Stack[T] squad {
    items []T
    size normie
}

fr fr Push method for any type T
slay push[T](s @Stack[T], item T) {
    s.items = append(s.items, item)
    s.size++
}

fr fr Pop method for any type T
slay pop[T](s @Stack[T]) T {
    lowkey s.size <= 0 {
        panic("stack is empty, that's not it chief")
    }
    s.size--
    yolo s.items[s.size]
}

Using Generic Types

fr fr Create stacks for different types
slay main() {
    fr fr Integer stack
    numStack := Stack[normie]{items: make([]normie, 0), size: 0}
    push(&numStack, 42)
    push(&numStack, 69)
    push(&numStack, 420)
    vibez.spill(pop(&numStack))  fr fr Outputs: 420
    
    fr fr String stack
    wordStack := Stack[tea]{items: make([]tea, 0), size: 0}
    push(&wordStack, "sheesh")
    push(&wordStack, "yeet")
    push(&wordStack, "slay")
    vibez.spill(pop(&wordStack))  fr fr Outputs: slay
}

CURSED also supports type constraints, letting you restrict what types can be used with your generic functions. It's like having a VIP-only area for your types.

be_like Numeric collab {
    add(other Numeric) Numeric
}

slay sum[T Numeric](a, b T) T {
    yolo a.add(b)
}

Pro tip that ate and left no crumbs

When naming type parameters, use single uppercase letters (like T, U, V) for simple cases, or more descriptive names like Element or Key for complex scenarios.
