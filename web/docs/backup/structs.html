Structs & Methods

Structs in CURSED are literally the moment! They let you create your own custom data types that group related data. And with methods? That's main character behavior.

Defining Structs
fr fr Basic struct definition
be_like Person squad {
    name tea
    age normie
    verified lit
}

fr fr Nested structs
be_like Address squad {
    street tea
    city tea
    state tea
    zip tea
}

be_like User squad {
    profile Person
    address Address
    createdAt timez.Time
}

Creating and Using Structs

slay main() {
    fr fr Creating a new struct instance
    bob := Person{
        name: "Bob",
        age: 25,
        verified: based,
    }
    
    fr fr Accessing fields
    vibez.spillf("Name: %s, Age: %d", bob.name, bob.age)
    
    fr fr Updating fields
    bob.age = 26
    
    fr fr Struct initialization shorthand (order matters)
    alice := Person{"Alice", 30, based}
    
    fr fr Creating nested structs
    user := User{
        profile: Person{"Charlie", 22, sus},
        address: Address{
            street: "123 Main St",
            city: "Los Angeles",
            state: "CA",
            zip: "90001",
        },
        createdAt: timez.Now(),
    }
}
Methods
fr fr Method with value receiver
slay (p Person) getFullName() tea {
    yolo p.name
}

fr fr Method with pointer receiver (can modify the struct)
slay (p @Person) celebrateBirthday() {
    p.age++  fr fr This actually modifies the original struct
}

slay (p @Person) setVerified(status lit) {
    p.verified = status
}

fr fr Using methods
slay main() {
    person := Person{"Taylor", 33, based}
    
    fr fr Call methods
    name := person.getFullName()
    vibez.spill("Name:", name)
    
    fr fr Call method that modifies the struct
    person.celebrateBirthday()
    vibez.spill("New age:", person.age)  fr fr 34
    
    fr fr Call another method with parameters
    person.setVerified(sus)
    vibez.spill("Verified:", person.verified)  fr fr false
}

Anonymous Structs - Giving temporary tea

When you need a struct for just a moment, no need to define a whole type:

fr fr Using anonymous struct
config := squad {
    endpoint tea
    timeout normie
    retries normie
}{
    endpoint: "https://api.example.com",
    timeout: 30,
    retries: 3,
}

vibez.spill(config.endpoint)

When to use pointer receivers

The tea on when to use value vs pointer receivers:

  • Use pointer receivers (@Person) when the method needs to change the struct
  • Use pointer receivers when the struct is large to avoid copying
  • Use value receivers (Person) when the method only needs to read the struct
  • Use value receivers for immutable data or small structs
  • Be consistent! If one method uses a pointer receiver, consider using it for all methods

Struct Tags

Add metadata to your struct fields to serve different purposes:

be_like User squad {
    username tea `json:"user_name" validate:"required"`
    email tea `json:"email,omitempty" validate:"email"`
    password tea `json:"-" validate:"min=8"`  fr fr Dash means don't include in JSON
}

fr fr These tags can be used by libraries for serialization, validation, etc.
