LLVM IR: The Secret Sauce

LLVM IR (Intermediate Representation) is the language that sits between CURSED and machine code. It's like the universal translator that helps CURSED speak fluently to any hardware.

What Is LLVM IR?

LLVM IR is a low-level programming language that's:

  • Typed - Every value has a well-defined type
  • SSA-based (Static Single Assignment) - Each variable is assigned exactly once
  • Platform-independent - The same IR works across different architectures
  • Human-readable (kinda) - You can actually read and debug it

When CURSED compiles your code, it translates your vibe-checking slang into this more formal language that LLVM can optimize and turn into machine code.

  call void @vibez.spill(...)
  br label %end

else:                      ; small vibe block

String Switch Implementation

        showHelp()
    mood "exit":
        exitApp()

Debug Information

CURSED embeds debug information in the LLVM IR to enable source-level debugging:

Debug Metadata
; LLVM Debug Info for a function

!10 = distinct !DISubprogram(name: "add", line: 42, isLocal: false, ...)
!12 = !DILocation(line: 43, column: 10, scope: !10)
!13 = !DILocation(line: 43, column: 3, scope: !10)

This allows the debugger to map machine code back to CURSED source lines and let you step through your code in an IDE or debugger.
