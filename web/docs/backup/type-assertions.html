Type Assertions

Type assertions in CURSED let you extract or check the concrete type behind an interface. It's giving type detective energy when you have dynamic values but need to know their actual identity.

Basic Type Assertion
fr fr Create a variable with an interface type
sus i collab{} = "hello world"  fr fr i holds a string value

fr fr Type assertion to extract the concrete type
sus s tea = i.(tea)  fr fr Extract the string value
vibez.spill(s)  fr fr "hello world"

fr fr This would panic if i doesn't contain a string
fr fr j := 42
fr fr i = j
fr fr s = i.(tea)  // PANIC: interface conversion

Type Assertion with Check

str, ok := i.(tea)  fr fr Try to extract a string

lowkey ok {
    vibez.spill("The value is a string:", str)
} highkey {
    vibez.spill("The value is not a string")
}

fr fr Now try the correct type
num, ok := i.(normie)  fr fr Try to extract an int

lowkey ok {
    vibez.spill("The value is an int:", num)
} highkey {
    vibez.spill("The value is not an int")
}
Type Switch
fr fr Type switch for checking multiple possible types
slay describeValue(v collab{}) {
    vibe_check v.(be_like) {
    mood normie:
        num := v.(normie)
        vibez.spill("It's an integer:", num)
        
    mood tea:
        str := v.(tea)
        vibez.spill("It's a string with length", len(str))
        
    mood []collab{}:
        slice := v.([]collab{})
        vibez.spill("It's a slice with", len(slice), "elements")
        
    mood lit:
        b := v.(lit)
        vibez.spill("It's a boolean:", b)
        
    basic:
        vibez.spill("It's some other type:", v)
    }
}

fr fr Try with different values
describeValue(42)        fr fr "It's an integer: 42"
describeValue("hello")   fr fr "It's a string with length 5"
describeValue(based)     fr fr "It's a boolean: true"
describeValue([]normie{1, 2, 3})  fr fr "It's a slice with 3 elements"

Interface Type Assertions vs. Concrete Types

Type assertions only work with interface types, not concrete types:

fr fr Works - interface to concrete
sus i collab{} = "hello"
str := i.(tea)  fr fr Extracting concrete type tea from interface

fr fr Won't compile - can't assert between concrete types
fr fr sus n normie = 42
fr fr f := n.(snack)  // Compile error: not an interface

fr fr Works - asserting interface implementation
be_like Stringer collab {
    String() tea
}

sus obj MyType = MyType{}
sus s Stringer = obj

fr fr Later, check if s implements another interface
writer, ok := s.(Writer)  fr fr Check if s also implements Writer
Working with Custom Types
fr fr Define some interface and types
be_like Shape collab {
    Area() snack
}

be_like Circle squad {
    radius snack
}

be_like Rectangle squad {
    width, height snack
}

fr fr Implement Shape interface for Circle
slay (c Circle) Area() snack {
    yolo 3.14 * c.radius * c.radius
}

fr fr Implement Shape interface for Rectangle
slay (r Rectangle) Area() snack {
    yolo r.width * r.height
}

fr fr Function using type assertions
slay describeShape(s Shape) {
    vibe_check s.(be_like) {
    mood Circle:
        c := s.(Circle)
        vibez.spillf("Circle with radius %.2f\n", c.radius)
        
    mood Rectangle:
        r := s.(Rectangle)
        vibez.spillf("Rectangle with width %.2f and height %.2f\n", 
                     r.width, r.height)
        
    basic:
        vibez.spill("Unknown shape type")
    }
    
    vibez.spillf("Area: %.2f\n", s.Area())
}

Type Assertion Best Practices

  • Always use the value, ok := x.(T) form to avoid panics
  • Use type switches for multiple possible types rather than chained if statements
  • Return early when a specific type is expected but not found
  • Design functions to accept interfaces rather than concrete types
  • Minimize the need for type assertions by designing clear interfaces

Empty Interface and Type Assertions

The empty interface collab{} can hold any value, but requires type assertions to use that value in a type-specific way:

fr fr Function that takes any type
slay formatAny(value collab{}) tea {
    fr fr Attempt to format based on type
    vibe_check v := value.(be_like) {
    mood normie:
        yolo vibez.spillstr("Number: %d", v)
        
    mood snack, meal:
        yolo vibez.spillstr("Float: %.2f", v)
        
    mood tea:
        yolo vibez.spillstr("String: %q", v)
        
    mood []collab{}:
        yolo vibez.spillstr("Slice with %d items", len(v))
        
    mood tea[tea]collab{}:
        yolo vibez.spillstr("Map with %d entries", len(v))
        
    basic:
        yolo vibez.spillstr("Unknown type: %v", v)
    }
}
