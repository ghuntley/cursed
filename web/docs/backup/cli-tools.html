Reference Tools Command Line

The CURSED programming language ships with a set of command-line tools that are giving main character energy for your development workflow.

The CURSED Command

The main cursed command is your bestie for all CURSED-related tasks:

Basic Usage
$ cursed [command] [options] [arguments]

# Examples
$ cursed run hello.csd         # Run a program
$ cursed build -o app main.csd # Build an executable
$ cursed test ./...           # Run all tests
$ cursed                      # Start REPL

Core Commands

Command Description Example
run Compile and run a CURSED program cursed run app.csd
build Compile a program without running it cursed build -o myapp main.csd
test Run tests in the current directory cursed test
fmt Format CURSED source files cursed fmt *.csd
doc Generate documentation cursed doc package
get Download and install packages cursed get github.com/user/pkg
version Show the CURSED version cursed version

The Run Command

The run command is your go-to for quick execution:

cursed run
$ cursed run [options] main-file.csd [arguments]

Options:
  -o, --output NAME     Set output file name
  -v, --verbose         Enable verbose output
  -r, --race            Enable race condition detection
  -d, --debug           Include debug information
  -O, --optimize LEVEL  Set optimization level (0-3)

Examples:

$ cursed run hello.csd                     # Run hello.csd
$ cursed run -race server.csd             # Run with race detection
$ cursed run app.csd arg1 arg2            # Pass arguments to program
$ cursed run -O2 -o myapp main.csd        # Optimize with level 2

The Build Command

Use build to compile your program without running it:

cursed build
$ cursed build [options] main-file.csd

Options:
  -o, --output NAME     Set output file name
  -v, --verbose         Enable verbose output
  -d, --debug           Include debug information
  -O, --optimize LEVEL  Set optimization level (0-3)
  -s, --static          Create statically linked binary
  -c, --cross TARGET    Cross-compile for target platform

Examples:

$ cursed build -o app main.csd              # Build to "app"
$ cursed build -O3 -o release.exe main.csd  # Max optimization
$ cursed build -s app.csd                  # Static linking
$ cursed build -c "windows/amd64" app.csd   # Cross-compile

The Test Command

Run tests with flexible path options:

cursed test
$ cursed test [options] [packages]

Options:
  -v, --verbose         Show verbose test output
  -r, --race            Enable race condition detection
  -t, --timeout DURATION  Set test timeout (e.g., "30s")
  -b, --bench           Run benchmarks instead of tests
  -f, --filter PATTERN  Run only tests matching pattern
  --cover               Show test coverage

Examples:

$ cursed test                                # Test current package
$ cursed test ./...                          # Test all subpackages
$ cursed test -v crypto/                     # Verbose test for crypto
$ cursed test -f "Test*Parser" ./parser/      # Filter tests
$ cursed test --cover                        # Show coverage
$ cursed test -b                             # Run benchmarks

The Format Command

Keep your code aesthetic with the formatter:

cursed fmt
$ cursed fmt [options] [files]

Options:
  -l, --list            List files that would be modified
  -w, --write           Write changes to files
  -d, --diff            Display diffs instead of rewriting
  -s, --simplify        Simplify code where possible

Examples:

$ cursed fmt -l *.csd                   # Check which files need formatting
$ cursed fmt -w .                       # Format all files in current dir
$ cursed fmt -d file.csd                # Show changes without applying
$ cursed fmt -w -s legacy.csd           # Format and simplify

The REPL (Interactive Shell)

Running cursed without arguments starts the REPL (Read-Eval-Print Loop):

CURSED REPL
$ cursed

Type "help" for assistance

> bestie i := 0; i < 3; i++ { vibez.spill(i) }

Package Management

Manage dependencies with the get, list, and update commands:

cursed get
$ cursed get [options] [packages]

Options:
  -u, --update          Update named packages
  -d, --dev             Install as development dependency
  -v, --version VERSION Specify version constraint
Package Management Examples
$ cursed get github.com/user/coolpkg        # Install package
$ cursed get -u ./...                       # Update all dependencies
$ cursed get -v "v1.2.3" some/package       # Get specific version
$ cursed list                               # List installed packages
$ cursed update                             # Update all packages

Tool Configuration

CURSED tools can be configured globally or per-project:

cursed.config.json
{
  "build": {
    "optimize": 2,
    "debug": true,
    "tags": ["dev", "testing"]
  },
  "format": {
    "indent": 4,
    "maxWidth": 100
  },
  "test": {
    "timeout": "30s",
    "coverageGoal": 80
  },
  "lint": {
    "rules": ["naming", "imports", "errors"],
    "exclude": ["vendor/*"]
  }
}

This config file can be placed in:

  • �� Project root as cursed.config.json for project-specific settings
  • �� ~/.cursed/config.json for user-wide global settings

Environment Variables

CURSED respects these environment variables:

Variable Purpose Example
CURSED_PATH Where to find packages export CURSED_PATH=~/projects/cursed-libs
CURSED_CACHE Cache directory location export CURSED_CACHE=/tmp/cursed-cache
CURSED_OPTIMIZE Default optimization level export CURSED_OPTIMIZE=2
CURSED_DEBUG Enable debug by default export CURSED_DEBUG=1
CURSED_TIMEOUT Default test timeout export CURSED_TIMEOUT=60s

CI/CD Integration

CURSED works great in continuous integration environments:

.github/workflows/cursed.yml
name: CURSED CI

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Set up CURSED
        uses: cursed-actions/setup@v1
        with:
          version: '1.x'
      
      - name: Build
        run: cursed build ./...
      
      - name: Test
        run: cursed test --cover ./...
      
      - name: Lint
        run: cursed lint ./...
