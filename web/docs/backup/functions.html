Core Concept Code Organization Fundamentals

Functions in CURSED literally slay! They're the main character of your code, doing all the hard work while looking absolutely fab. Let's spill the tea on how to use them.

Function Basics

Functions are blocks of reusable code that perform specific tasks. In CURSED, you define them using the slay keyword.

Basic Function
fr fr Function with parameters and return type
slay add(x normie, y normie) normie {
    yolo x + y  fr fr Return the sum
}

fr fr Function with no parameters or return value
slay sayHi() {
    vibez.spill("Hiiii bestie!")
}

fr fr Function with multiple return values
slay divideAndRemainder(a normie, b normie) (normie, normie) {
    yolo a / b, a % b
}

Calling Functions

Using functions is straightforward - just call them by name with the appropriate arguments:

Function Calls
fr fr Basic function calls
sayHi()  fr fr Calls function with no arguments

sus result = add(5, 3)  fr fr Calls add function with arguments 5 and 3
vibez.spill(result)  fr fr Output: 8

fr fr Using multiple return values
sus quotient, remainder = divideAndRemainder(10, 3)
vibez.spill(quotient, remainder)  fr fr Output: 3 1

fr fr Ignoring values you don't need with _
sus _, rem = divideAndRemainder(10, 3)  fr fr Only keep remainder

Named Return Values

You can name return values for clarity and implicit returns:

Named Return Values
fr fr Named return values
slay divide(a normie, b normie) (quotient normie, remainder normie) {
    quotient = a / b
    remainder = a % b
    yolo  fr fr Empty return uses named values
}

fr fr Named returns with early exit
slay safeDivide(a normie, b normie) (q normie, r normie, err tea) {
    lowkey b == 0 {
        err = "Cannot divide by zero"
        yolo  fr fr Returns zero values for q and r, with error message
    }
    
    q = a / b
    r = a % b
    yolo  fr fr Returns q, r, and empty string for err
}

Variadic Functions

Functions that accept a variable number of arguments:

Variadic Functions
fr fr Variadic function with the ... syntax
slay sum(numbers ...normie) normie {
    sus total normie = 0
    bestie _, num := flex numbers {
        total += num
    }
    yolo total
}

fr fr Calling variadic functions
vibez.spill(sum(1, 2, 3))  fr fr Output: 6
vibez.spill(sum(5, 10, 15, 20))  fr fr Output: 50
vibez.spill(sum())  fr fr Output: 0 (no arguments is valid)

fr fr Spreading a slice into variadic arguments
nums := []normie{1, 2, 3, 4, 5}
vibez.spill(sum(nums...))  fr fr Output: 15

Anonymous Functions

Create functions on the fly without naming them:

Anonymous Functions
fr fr Anonymous function assigned to a variable
sus double = slay(x normie) normie {
    yolo x * 2
}

vibez.spill(double(5))  fr fr Output: 10

fr fr Anonymous function as an argument
slay apply(n normie, f slay(normie) normie) normie {
    yolo f(n)
}

vibez.spill(apply(5, double))  fr fr Output: 10

fr fr Inline anonymous function
result := apply(5, slay(x normie) normie {
    yolo x * x  fr fr Square the input
})
vibez.spill(result)  fr fr Output: 25

Closures

Anonymous functions that capture variables from their surrounding scope:

Closures
fr fr Function that returns a closure
slay multiplier(factor normie) slay(normie) normie {
    fr fr Returns a function that multiplies by factor
    yolo slay(x normie) normie {
        yolo x * factor  fr fr Captures 'factor' from outer scope
    }
}

fr fr Create specialized multiplier functions
sus double = multiplier(2)
sus triple = multiplier(3)

vibez.spill(double(5))  fr fr Output: 10
vibez.spill(triple(5))  fr fr Output: 15
