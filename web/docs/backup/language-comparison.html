Migration Paths

Curious about moving existing code to CURSED? Here's how it maps:

From Go to CURSED

Easiest transition - just replace keywords with their CURSED equivalents, adjust package syntax, and you're mostly there.

Difficulty: Easy

From Python to CURSED

Requires adding type annotations and adjusting to block-based syntax. Package structure changes are needed.

Difficulty: Moderate

From JavaScript to CURSED

Requires adapting to static typing, changing event-based concurrency to goroutines, and restructuring modules.

Difficulty: Moderate

From Rust to CURSED

Need to remove ownership annotations, replace Result/Option with explicit error checking, and adapt to GC.

Difficulty: Hard
