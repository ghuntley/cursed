✨ Reflection Basics ✨

Reflection lets you examine types, fields, and methods during runtime:

    // Get the runtime value representation
    sus xValue = reflectz.ValueOf(x);
    vibez.spill("Value of x is", xValue.Int());  // 42
    
    // Check if it's a basic type

    // Get a string representation
    sus str = "hello";
    vibez.spill("Type of str is", reflectz.TypeOf(str).Name());  // "tea" or "string"

📲 Modifying Values 📲

To modify values through reflection, you need to work with pointers:

modify_values.csd
slay main() {
    sus p = Person{Name: "Alex", Age: 25, IsAdmin: cap};
    
    // Create a reflection Value pointing to p
    sus ptrValue = reflectz.ValueOf(&p);  // Note the &
    
    // Get the value that the pointer points to
    sus elemValue = ptrValue.Elem();
    
    // Now we can modify fields
    sus nameField = elemValue.FieldByName("Name");
    lowkey nameField.CanSet() {  // Always check if it's settable
        nameField.SetString("Alex (Modified)");
    }
    
    sus ageField = elemValue.FieldByName("Age");
    lowkey ageField.CanSet() {
        ageField.SetInt(26);  // Happy birthday!
    }
    
    vibez.spill("Modified struct:", p);
}

The Tea on Settability ��

A reflection Value is only settable if:

  • It was created from a pointer and dereferenced with Elem()
  • The field is exported (starts with uppercase)
  • It's not coming from an unexported struct field

Always check CanSet() before trying to modify a value!

Bestie Advice ��

��

Use With Caution

Reflection is slower than direct code and bypasses type safety. Only use it when you really need runtime type information.

��

Check All Errors

Always check IsValid() and CanSet() before using reflection values to avoid panics.

👻

Mind The Performance

Reflection operations are much slower than direct code. Cache results when possible.

��‍��

Consider Alternatives

Often interfaces, generics, or code generation can provide type-safe alternatives to reflection.

FAQ - Tea Time ☕

When should I use reflection?

Reflection is mostly useful for:

  • Generic serialization/deserialization (JSON, XML, etc.)
  • Dynamic method dispatch or plugin systems
  • Dependency injection frameworks
  • Testing utilities that need to analyze code
  • ORM implementations

But if you can solve your problem with interfaces or generics, that's usually better!

Can I create new types at runtime?

No, CURSED doesn't allow creating new types at runtime. Reflection allows examining existing types and manipulating values, but you can't define brand new types dynamically.

Why is my field not settable?

Fields might not be settable for several reasons:

  • You're working with a value, not a pointer (use &obj and Elem())
  • The field starts with lowercase (unexported fields can't be set)
  • The field is inside an unexported embedded struct
  • You're working with a map or array element (use specialized Set methods)
