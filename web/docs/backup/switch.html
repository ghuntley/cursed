Switch Statements

When you've got multiple vibes to check, the vibe_check statement is your bestie. It's giving cleaner code and better organization than a bunch of lowkey/highkey statements.

Basic Switch
fr fr Switch on a value
sus day tea = "Wednesday"

vibe_check day {
mood "Monday":
    vibez.spill("Start of work week")
mood "Tuesday", "Wednesday", "Thursday":
    vibez.spill("Midweek grind")
mood "Friday":
    vibez.spill("TGIF energy")
mood "Saturday", "Sunday":
    vibez.spill("Weekend vibes")
basic:
    vibez.spill("Not a valid day")
}

Type Switch

Check the type of an interface value - it's giving dynamic typing vibes:

slay describeValue(v collab{}) {
    fr fr Switch on the type of v
    vibe_check v.(be_like) {
    mood normie:
        num := v.(normie)  fr fr Type assertion after knowing the type
        vibez.spill("It's an integer with value:", num)
    mood tea:
        text := v.(tea)
        vibez.spill("It's a string with value:", text)
        vibez.spillf("The string has %d characters\n", len(text))
    mood []normie:
        nums := v.([]normie)
        vibez.spill("It's an integer slice with values:", nums)
        vibez.spill("The slice has", len(nums), "elements")
    mood tea[tea]collab{}:
        dict := v.(tea[tea]collab{})
        vibez.spill("It's a map with", len(dict), "entries")
    basic:
        vibez.spill("Unknown type with value:", v)
    }
}

slay main() {
    describeValue(42)
    describeValue("hello")
    describeValue([]normie{1, 2, 3})
    describeValue(tea[tea]collab{}{"key": "value"})
    describeValue(based)
}
Type Switch with Assignment
slay getValue(key tea) collab{} {
    fr fr Function returns interface{}
    yolo "hello"  fr fr Example return value
}

slay main() {
    fr fr Type switch with variable assignment
    vibe_check value := getValue("someKey"); value.(be_like) {
    mood normie:
        fr fr 'value' is assigned to the matched type
        vibez.spillf("Got integer: %d\n", value)
    mood tea:
        fr fr 'value' here is of type string
        vibez.spillf("Got string: %s (length: %d)\n", value, len(value))
    mood lit:
        fr fr 'value' here is of type bool
        vibez.spillf("Got boolean: %t\n", value)
    basic:
        vibez.spillf("Got unknown type: %v\n", value)
    }
}

Switch Fallthrough Behavior

Unlike some languages, CURSED switches don't fall through to the next case by default. Each case automatically breaks. If you want fallthrough behavior, use the fallthrough keyword:

sus num normie = 5

vibe_check num {
mood 1:
    vibez.spill("One")
mood 5:
    vibez.spill("Five")
    fallthrough  fr fr Explicitly fall through to next case
mood 10:
    vibez.spill("Ten or fallthrough from Five")
basic:
    vibez.spill("Other number")
}

Switch vs If-Else

Switch If-Else
Better for multiple related conditions Better for few unrelated conditions
More readable with many cases More flexible for complex conditions
Cases must be constants or expressions Can use any boolean expression
No implicit fallthrough N/A
Great for type checking with interfaces Requires explicit type assertions
