Interfaces

Interfaces in CURSED are all about collab energy! They define a set of methods that types must implement to join the squad. It's giving modular design vibes and loose coupling.

Interface Definition
fr fr Define a basic interface
be_like Shape collab {
    area() snack
    perimeter() snack
}

Implementing Interfaces

In CURSED, types implement interfaces implicitly - if they have the right methods, they're automatically part of the collab:

fr fr Define types that implement the Shape interface
be_like Rectangle squad {
    width snack
    height snack
}

slay (r Rectangle) area() snack {
    yolo r.width * r.height
}

slay (r Rectangle) perimeter() snack {
    yolo 2 * (r.width + r.height)
}

be_like Circle squad {
    radius snack
}

slay (c Circle) area() snack {
    yolo mathz.Pi * c.radius * c.radius
}

slay (c Circle) perimeter() snack {
    yolo 2 * mathz.Pi * c.radius
}
Using Interfaces
fr fr Function that accepts any Shape
slay printShapeInfo(s Shape) {
    vibez.spillf("Area: %f", s.area())
    vibez.spillf("Perimeter: %f", s.perimeter())
}

slay main() {
    rect := Rectangle{width: 5.0, height: 4.0}
    circle := Circle{radius: 3.0}
    
    fr fr Both can be used where a Shape is expected
    printShapeInfo(rect)
    printShapeInfo(circle)
    
    fr fr Create a slice of different shapes
    shapes := []Shape{
        Rectangle{width: 3.0, height: 4.0},
        Circle{radius: 5.0},
    }
    
    fr fr Iterate over shapes
    bestie _, shape := flex shapes {
        vibez.spillf("Shape area: %f", shape.area())
    }
}

Empty Interface

The empty interface collab{} can hold values of any type - it's the ultimate flexible container:

fr fr Function that accepts any value
slay describe(v collab{}) {
    vibe_check v.(be_like) {
    mood normie:
        vibez.spill("It's an integer:", v)
    mood tea:
        vibez.spill("It's a string:", v)
    mood lit:
        vibez.spill("It's a boolean:", v)
    basic:
        vibez.spill("It's something else:", v)
    }
}

slay main() {
    describe(42)
    describe("hello")
    describe(based)
    describe([]normie{1, 2, 3})
}

Interface Composition - Stack those collabs!

Interfaces can embed other interfaces to create larger interface contracts:

fr fr Simple interfaces
be_like Reader collab {
    read([]byte) (normie, tea)
}

be_like Writer collab {
    write([]byte) (normie, tea)
}

fr fr Composed interface
be_like ReadWriter collab {
    Reader
    Writer
}

fr fr Now any type must implement both read and write methods
Type Assertions
fr fr Getting the concrete type from an interface
slay main() {
    sus shape Shape = Circle{radius: 5.0}
    
    fr fr Type assertion with check
    circle, ok := shape.(Circle)
    lowkey ok {
        vibez.spillf("It's a circle with radius %f", circle.radius)
    }
    
    fr fr Type assertion without check (will panic if wrong)
    fr fr rectangle := shape.(Rectangle) // would panic here
    
    fr fr Type switch for multiple possibilities
    vibe_check shape.(be_like) {
    mood Circle:
        c := shape.(Circle)
        vibez.spill("Circle with area:", c.area())
    mood Rectangle:
        r := shape.(Rectangle)
        vibez.spill("Rectangle with area:", r.area())
    basic:
        vibez.spill("Unknown shape with area:", shape.area())
    }
}

Interface Design Best Practices

  • Keep interfaces small and focused (one or few methods)
  • Design interfaces based on behavior, not data
  • Accept interfaces, return concrete types
  • Use composition to build larger interfaces from smaller ones
  • Don't export interfaces for types that satisfy the interface
