Composite Types

Composite types in CURSED let you build more complex data structures by combining simpler types. It's giving organized storage vibes and letting you create exactly the data types your code needs.

Structs (squad)

fr fr Define a basic struct
be_like Person squad {
    name tea
    age normie
    isActive lit
}

fr fr Create a struct instance
person := Person{
    name: "Taylor",
    age: 25,
    isActive: based,
}

fr fr Access fields
vibez.spill(person.name)     fr fr "Taylor"
vibez.spill(person.age)      fr fr 25
vibez.spill(person.isActive) fr fr true

fr fr Modify fields
person.age = 26
person.isActive = sus
Embedded Structs
fr fr Define structs with embedding
be_like Address squad {
    street tea
    city tea
    zipCode tea
}

be_like User squad {
    fr fr Embed Person struct (anonymous field)
    Person
    
    fr fr Embed Address struct (anonymous field)
    Address
    
    fr fr Regular field
    email tea
}

fr fr Create a User
user := User{
    Person: Person{
        name: "Alex",
        age: 28,
        isActive: based,
    },
    Address: Address{
        street: "123 Main St",
        city: "Los Angeles",
        zipCode: "90001",
    },
    email: "alex@example.com",
}

fr fr Access fields directly (promoted fields)
vibez.spill(user.name)     fr fr From Person
vibez.spill(user.street)   fr fr From Address
vibez.spill(user.email)    fr fr Direct field

fr fr Alternative: explicit access
vibez.spill(user.Person.name)
vibez.spill(user.Address.street)

Arrays

fr fr Fixed-size array
sus numbers [5]normie = [5]normie{1, 2, 3, 4, 5}

fr fr Shorthand initialization
colors := [3]tea{"red", "green", "blue"}

fr fr Let compiler count the size
days := [...]tea{"Monday", "Tuesday", "Wednesday", "Thursday", "Friday"}

fr fr Access by index
firstColor := colors[0]  fr fr "red"

fr fr Get length
numDays := len(days)  fr fr 5

fr fr Multi-dimensional arrays
matrix := [3][3]normie{
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9},
}

fr fr Access element in 2D array
middle := matrix[1][1]  fr fr 5
Slices
fr fr Create a slice
fruits := []tea{"apple", "banana", "cherry"}

fr fr Create empty slice
numbers := []normie{}

fr fr Create slice with make
buffer := make([]byte, 10)  fr fr 10 bytes, all zeroes

fr fr Slice with capacity
queue := make([]normie, 0, 10)  fr fr Length 0, capacity 10

fr fr Append to slice
fruits = append(fruits, "date")
fruits = append(fruits, "elderberry", "fig")

fr fr Slicing
firstTwo := fruits[:2]       fr fr ["apple", "banana"]
middle := fruits[1:4]        fr fr ["banana", "cherry", "date"]
rest := fruits[3:]           fr fr ["date", "elderberry", "fig"]
copy := fruits[:]            fr fr Same as fruits

fr fr Copy slices
dest := make([]tea, len(fruits))
copied := copy(dest, fruits)  fr fr Returns number of elements copied

Maps

fr fr Declare map (string keys, int values)
sus scores tea[tea]normie

fr fr Initialize with make
scores = make(tea[tea]normie)

fr fr Initialize with values
colors := tea[tea]tea{
    "red": "#ff0000",
    "green": "#00ff00",
    "blue": "#0000ff",
}

fr fr Set key-value pairs
scores["alice"] = 95
scores["bob"] = 87

fr fr Get values
aliceScore := scores["alice"]  fr fr 95

fr fr Check if key exists
score, exists := scores["charlie"]  fr fr score=0, exists=false

fr fr Delete a key
delete(scores, "bob")

fr fr Iterate over a map
bestie name, score := flex scores {
    vibez.spillf("%s scored %d\n", name, score)
}

Advanced Struct Patterns

fr fr Struct Tags for metadata
be_like User squad {
    ID        normie `json:"id" db:"user_id"`
    Name      tea   `json:"name" validate:"required"`
    Email     tea   `json:"email,omitempty" validate:"email"`
    Password  tea   `json:"-" db:"password_hash"`  fr fr - means skip in JSON
}

Struct tags are string literals that provide metadata about the field. They're used by external packages for:

  • JSON/XML serialization
  • Database column mapping
  • Form validation
  • CLI flags
Nested Composite Types

be_like Post squad {
    title     tea

    text      tea

post := Post{

}

Which Composite Type to Use When

Type When to Use Example Use Case
Arrays Fixed-size collections of same type Buffer of known size, coordinate points
Slices Dynamic-size collections of same type List of users, search results, items to process
Maps Key-value lookups User data by ID, configuration settings, caches
Structs Grouping related data with named fields User profiles, database records, complex objects
Nested Complex hierarchical data JSON/XML data structures, tree-like structures

Type Definitions vs Type Aliases

fr fr Type definition (creates a new type)
be_like UserID normie

fr fr Type alias (just an alternative name)
be_like Score = normie

slay main() {
    fr fr UserID is a distinct type
    sus id UserID = 1001
    sus num normie = 1001
    
    fr fr This would be a type error:
    fr fr id = num  // Cannot assign normie to UserID
    
    fr fr Need explicit conversion
    id = UserID(num)  fr fr OK
    
    fr fr Score is just an alias, freely interchangeable
    sus s Score = 95
    num = s       fr fr OK, no conversion needed
    s = num       fr fr Also OK
}
