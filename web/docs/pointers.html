Pointers

Pointers in CURSED are giving reference energy! Instead of copying data, they let you directly reference memory locations. It's the ultimate way to share and modify values with maximum efficiency.

Pointer Basics
fr fr Create a variable
sus x normie = 42

fr fr Create a pointer to x with @
sus p @normie = @x  fr fr p stores the memory address of x

fr fr Dereference the pointer to get the value
value := *p  fr fr value = 42

fr fr Modify the value through the pointer
*p = 100  fr fr Now x = 100

fr fr Check that x changed
vibez.spill("x is now:", x)  fr fr Outputs: x is now: 100

Pointer Types

The pointer type is denoted by @ followed by the base type:

Type Description Example
@normie Pointer to an integer sus p @normie
@tea Pointer to a string sus p @tea
@Person Pointer to a Person struct sus p @Person
@@normie Pointer to a pointer to an integer sus pp @@normie
@[]tea Pointer to a slice of strings sus p @[]tea
Pointers with Structs
fr fr Define a struct
be_like Person squad {
    name tea
    age normie
}

fr fr Create a struct
sus person Person = Person{name: "Taylor", age: 25}

fr fr Create a pointer to the struct
sus personPtr @Person = @person

fr fr Access fields through the pointer
vibez.spill(personPtr.name)  fr fr "Taylor" (no need for *personPtr.name)

fr fr Modify fields through the pointer
personPtr.age = 26

fr fr The original struct is modified
vibez.spill(person.age)  fr fr 26

Function Parameters

fr fr Pass by value (makes a copy)
slay incrementValue(x normie) {
    x++  fr fr Only modifies the local copy
}

fr fr Pass by reference (uses pointer)
slay incrementPointer(x @normie) {
    *x++  fr fr Modifies the original value
}

slay main() {
    num := 10
    
    incrementValue(num)
    vibez.spill("After incrementValue:", num)  fr fr Still 10
    
    incrementPointer(@num)
    vibez.spill("After incrementPointer:", num)  fr fr Now 11
}

Watch Out For These Pointer Gotchas

fr fr Nil pointer dereference
sus ptr @normie  fr fr Default zero value is nil
fr fr *ptr = 42  // This would panic at runtime!

fr fr Check for nil before dereferencing
lowkey ptr != cap {
    *ptr = 42
} highkey {
    vibez.spill("Cannot dereference nil pointer")
}

fr fr Cannot take the address of a temporary value
fr fr invalidPtr := @(10 * 2)  // Compilation error

fr fr Fix by using a variable
value := 10 * 2
validPtr := @value  fr fr This works
Pointer Arithmetic
fr fr CURSED doesn't allow direct pointer arithmetic like C/C++
fr fr You can't do: ptr++ or ptr + 1

fr fr For array/slice traversal, use indexes instead
arr := [5]normie{1, 2, 3, 4, 5}
bestie i := 0; i < len(arr); i++ {
    ptr := @arr[i]  fr fr Get pointer to each element
    *ptr *= 2       fr fr Double each element
}

fr fr For multiple indirection levels, dereference step by step
sus x normie = 10
sus p @normie = @x
sus pp @@normie = @p

fr fr Access the value through multiple dereferences
value := **pp  fr fr value = 10

fr fr Modify the value through multiple dereferences
**pp = 20      fr fr Now x = 20

When to Use Pointers

  • Modifying Arguments: When a function needs to modify its parameters
  • Large Structs: To avoid copying large data structures
  • Shared Ownership: When multiple parts of code need to access the same data
  • Nullable Values: When you need to represent a value that might not exist
  • Implementing Methods: For methods that modify the receiver

Pointer Receivers in Methods

be_like Counter squad {
    value normie
}

fr fr Value receiver - doesn't modify the original
slay (c Counter) getValue() normie {
    yolo c.value
}

fr fr Pointer receiver - modifies the original
slay (c @Counter) increment() {
    c.value++  fr fr Equivalent to (*c).value++
}

slay main() {
    counter := Counter{value: 0}
    
    fr fr Call the method with pointer receiver
    counter.increment()  fr fr No need for (&counter).increment()
    counter.increment()
    
    fr fr The value is modified
    vibez.spill("Counter value:", counter.getValue())  fr fr 2
}
Creating New Struct Instances with New
fr fr Create a new zeroed instance of a struct and return a pointer
slay main() {
    fr fr Using new() to create a zeroed Person
    p := new(Person)  fr fr Returns a *Person
    
    fr fr Set fields
    p.name = "Alex"
    p.age = 30
    
    fr fr Alternatively, use literal with &
    p2 := @Person{name: "Jordan", age: 28}
    
    fr fr Both p and p2 are pointers to Person
    vibez.spill(p.name)   fr fr "Alex"
    vibez.spill(p2.name)  fr fr "Jordan"
}
