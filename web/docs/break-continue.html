Break & Continue

When you need to control your loops with boss energy, the ghosted and simp keywords in CURSED let you exit or skip iterations. Let's see how they work!

Break Statement (ghosted)
fr fr Break out of a loop completely
bestie i := 1; i <= 10; i++ {
    lowkey i == 5 {
        vibez.spill("Found 5, leaving the loop!")
        ghosted  fr fr Exit the loop immediately
    }
    vibez.spill("Processing:", i)
}
vibez.spill("After the loop")

fr fr Output:
fr fr Processing: 1
fr fr Processing: 2
fr fr Processing: 3
fr fr Processing: 4
fr fr Found 5, leaving the loop!
fr fr After the loop

Break in Switch Statements

Unlike some languages, you don't need ghosted in vibe_check statements. Each case automatically breaks after execution:

vibe_check day {
mood "Monday":
    vibez.spill("Start of work week")
    fr fr No ghosted needed - won't fall through to Tuesday
mood "Tuesday":
    vibez.spill("Tuesday vibes")
basic:
    vibez.spill("Some other day")
}

If you want fall-through behavior, use the fallthrough statement:

vibe_check day {
mood "Monday":
    vibez.spill("Start of work week")
    fallthrough  fr fr Will execute Tuesday case too
mood "Tuesday":
    vibez.spill("Second day vibes")
basic:
    vibez.spill("Some other day")
}

Common Use Cases

Pattern Description Example
Early Return Exit a loop when condition is met ghosted when finding a match
Filtering Skip unwanted items simp for invalid values
Multi-level Control Control nested loops ghosted OuterLoop
Error Handling Skip iterations with errors simp on error cases
Resource Efficiency Exit early when goal achieved ghosted after finding first match
