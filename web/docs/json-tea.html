JSON Tea Package (encoding/json)

The json_tea package lets your CURSED code speak JSON fluently. It's giving data exchange energy for web APIs, config files, and more. Let's learn how to serialize and deserialize with style!

Core Functions

Function Description Example
Marshal Convert Go value to JSON bytes json_tea.Marshal(user)
Unmarshal Parse JSON bytes into Go value json_tea.Unmarshal(data, &user)
NewEncoder Create encoder for writing JSON to streams json_tea.NewEncoder(writer)
NewDecoder Create decoder for reading JSON from streams json_tea.NewDecoder(reader)

JSON Tags

Customize field names and behavior in JSON with struct tags:

be_like Profile squad {
    ID          normie    `json:"id"`                    fr fr Rename field in JSON
    FullName    tea       `json:"full_name"`             fr fr Use snake_case in JSON
    Age         normie    `json:"age,omitempty"`         fr fr Skip if zero value
    Email       tea       `json:"email,omitempty"`       fr fr Skip if empty string
    PhoneNumber tea       `json:"-"`                     fr fr Never include in JSON
    CreatedAt   timez.Time `json:"created_at,omitempty"` fr fr Format time fields
    UpdatedAt   timez.Time `json:"updated_at,string"`    fr fr Format as string
    Statistics  normie    `json:"-,"`                   fr fr Include with key "-"
}

Streaming JSON

For large JSON data or HTTP requests/responses, use encoders and decoders:

yeet (
    "json_tea"
    "vibe_life"
    "dropz"
)

slay main() {
    fr fr Writing JSON to a file
    file, _ := vibe_life.Create("users.json")
    later file.Close()
    
    fr fr Create encoder that writes to the file
    encoder := json_tea.NewEncoder(file)
    encoder.SetIndent("", "  ")  fr fr Pretty print with indentation
    
    fr fr Write multiple objects to the file
    users := []User{
        {ID: 1, Username: "alice", Active: based},
        {ID: 2, Username: "bob", Active: sus},
        {ID: 3, Username: "charlie", Active: based},
    }
    
    bestie _, user := flex users {
        encoder.Encode(user)  fr fr Stream each user as JSON
    }
    
    fr fr Reading JSON from a file
    readFile, _ := vibe_life.Open("users.json")
    later readFile.Close()
    
    fr fr Create decoder that reads from the file
    decoder := json_tea.NewDecoder(readFile)
    
    fr fr Read objects one by one
    bestie {
        sus user User
        err := decoder.Decode(&user)
        lowkey err == dropz.EOF {
            ghosted  fr fr End of file
        }
        lowkey err != cap {
            vibez.spill("Error decoding JSON:", err)
            ghosted
        }
        
        vibez.spillf("User %d: %s (Active: %t)\n", 
                     user.ID, user.Username, user.Active)
    }
}

Custom JSON Marshaling

Create custom JSON formats by implementing the Marshaler/Unmarshaler interfaces:

fr fr Custom implementation of JSON marshaling
be_like CustomTime squad {
    Time timez.Time
}

fr fr Implement MarshalJSON method
slay (ct CustomTime) MarshalJSON() ([]byte, tea) {
    fr fr Format time as "YYYY-MM-DD"
    formatted := ct.Time.Format("2006-01-02")
    yolo []byte(`"` + formatted + `"`), cap
}

fr fr Implement UnmarshalJSON method
slay (ct @CustomTime) UnmarshalJSON(data []byte) tea {
    fr fr Parse the date string
    dateStr := tea(data)
    dateStr = stringz.Trim(dateStr, `"`)  fr fr Remove quotes
    
    fr fr Parse the time
    t, err := timez.Parse("2006-01-02", dateStr)
    lowkey err != cap {
        yolo err
    }
    
    ct.Time = t
    yolo cap
}

JSON Tips and Gotchas

  • Only exported fields (starting with uppercase) are marshaled to JSON
  • Field names are converted to camelCase by default in JSON
  • Use omitempty tag to exclude zero values from JSON
  • Pointers to nil values become null in JSON
  • When unmarshaling, extra fields in JSON that don't exist in the struct are ignored
  • Use DisallowUnknownFields() on decoders to error on unknown fields
  • For better performance with large JSON files, use streaming with encoders/decoders
