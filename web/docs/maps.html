Maps

Maps in CURSED are the ultimate key-value storage vibe! They let you associate keys with values, creating a lookup table that's giving dictionary energy. Let's get into how to use these flexible data structures.

Creating Maps
fr fr Declare a map with string keys and integer values
sus scores tea[tea]normie

fr fr Initialize with make
scores = make(tea[tea]normie)

fr fr Initialize with values (map literal)
sus userStatus = tea[tea]lit{
    "alice": based,
    "bob": sus,
    "charlie": based,
}

fr fr Initialize with capacity hint for better performance
userPrefs := make(tea[tea]tea, 100)  fr fr Initial capacity of 100

Map Operations

fr fr Set key-value pairs
scores["alice"] = 95
scores["bob"] = 82
scores["charlie"] = 88

fr fr Get values
aliceScore := scores["alice"]  fr fr 95

fr fr Check if a key exists
score, exists := scores["dave"]  fr fr score = 0 (zero value), exists = false
lowkey !exists {
    vibez.spill("Dave doesn't have a score yet")
}

fr fr Delete a key
delete(scores, "bob")  fr fr Remove bob's score

fr fr Get map length
numScores := len(scores)  fr fr 2 (alice and charlie)
Iterating Over Maps
fr fr Range over a map
bestie name, score := flex scores {
    vibez.spillf("%s scored %d\n", name, score)
}

fr fr Get just keys
bestie name := flex scores {
    vibez.spill("User:", name)
}

fr fr Note: map iteration order is not guaranteed!
fr fr Each run may produce different ordering

Maps as Sets

You can use maps to implement sets by using the keys as set members:

fr fr Create a set of strings
sus uniqueWords = make(tea[tea]lit)

fr fr Add to the set
uniqueWords["vibe"] = based
uniqueWords["slay"] = based
uniqueWords["bestie"] = based

fr fr Check if an item is in the set
lowkey uniqueWords["slay"] {
    vibez.spill("'slay' is in the set")
}

fr fr Remove from the set
delete(uniqueWords, "bestie")

fr fr Iterate over the set
bestie word := flex uniqueWords {
    vibez.spill("Word in set:", word)
}

Map Gotchas and Tips

Things to watch out for when working with maps:

fr fr Accessing a non-existent key returns zero value
sus counts = make(tea[tea]normie)
vibez.spill(counts["nonexistent"])  fr fr Prints 0

fr fr Safer way to access with existence check
count, exists := counts["nonexistent"]
lowkey !exists {
    count = -1  fr fr Default value for missing keys
}

fr fr Cannot take address of map element
fr fr This would cause a compilation error:
fr fr address := &counts["alice"]

fr fr Maps are reference types (passed by reference)
slay updateMap(m tea[tea]normie) {
    m["updated"] = 100  fr fr Modifies the original map
}
Nested Maps
fr fr Map of maps for complex data structures
sus userProfiles = make(tea[tea]tea[tea]tea)

fr fr Initialize nested maps
userProfiles["alice"] = tea[tea]tea{
    "email": "alice@example.com",
    "location": "New York",
    "status": "active",
}

userProfiles["bob"] = tea[tea]tea{
    "email": "bob@example.com",
    "location": "Los Angeles",
    "status": "inactive",
}

fr fr Access nested map data
aliceEmail := userProfiles["alice"]["email"]

fr fr Update nested data
userProfiles["alice"]["status"] = "away"

fr fr Check before access to avoid nil map panic
lowkey profile, exists := userProfiles["charlie"]; exists {
    vibez.spill(profile["email"])
} highkey {
    vibez.spill("Charlie not found")
}

Map Performance Considerations

  • Specify initial capacity when you know approximate size
  • Maps automatically grow as needed, but preallocating reduces rehashing
  • Maps have O(1) average time complexity for lookups, inserts, and deletes
  • Large maps with string keys benefit from string interning
  • Use composite keys carefully: consider a custom struct or string representation

Custom Key Types

Any comparable type can be used as a map key, including structs:

fr fr Define a struct to use as a key
be_like Coordinate squad {
    x, y normie
}

fr fr Create a map with Coordinate keys
sus grid = make(tea[Coordinate]tea)

fr fr Set values in the map
grid[Coordinate{10, 20}] = "treasure"
grid[Coordinate{15, 25}] = "monster"

fr fr Look up a value
playerPos := Coordinate{10, 20}
object := grid[playerPos]
vibez.spill("Found at player position:", object)
Thread Safety
fr fr Maps are NOT thread-safe!
fr fr For concurrent access, use a mutex

be_like SafeMap squad {
    data tea[tea]tea
    mutex concurrenz.RWMutex
}

slay NewSafeMap() @SafeMap {
    yolo &SafeMap{
        data: make(tea[tea]tea),
    }
}

slay (m @SafeMap) Set(key, value tea) {
    m.mutex.Lock()
    later m.mutex.Unlock()
    m.data[key] = value
}

slay (m @SafeMap) Get(key tea) (tea, lit) {
    m.mutex.RLock()
    later m.mutex.RUnlock()
    value, exists := m.data[key]
    yolo value, exists
}
