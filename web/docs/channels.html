Channels

Channels in CURSED are the way goroutines slide into each other's DMs! They let concurrent parts of your program pass messages with zero drama. It's giving synchronized communication energy.

Creating Channels
fr fr Create an unbuffered channel for integers
sus ch dm

fr fr Initialize it with make
ch = make(dm)  fr fr Unbuffered by default

fr fr Create a buffered channel that can hold 5 strings
messageCh := make(dm, 5)  fr fr Can store 5 messages without blocking

Sending and Receiving

fr fr Send a value into a channel
ch <- 42  fr fr Send 42 to the channel

fr fr Receive a value from a channel
value := <-ch  fr fr Read from the channel and store in value

fr fr Receive with an ok check for closed channels
value, ok := <-ch  fr fr ok will be false if channel is closed
Unbuffered Channels
slay main() {
    fr fr Create an unbuffered channel
    ch := make(dm)
    
    fr fr Start a goroutine that sends to the channel
    stan slay() {
        vibez.spill("Goroutine: about to send message")
        ch <- "hey bestie"  fr fr This blocks until someone receives
        vibez.spill("Goroutine: message sent")
    }()
    
    fr fr Give goroutine time to try to send
    timez.Sleep(1 * timez.Second)
    
    fr fr Receive the message (this unblocks the sender)
    vibez.spill("Main: about to receive")
    msg := <-ch
    vibez.spill("Main: received message:", msg)
}

Buffered Channels

slay main() {
    fr fr Create a buffered channel with capacity 3
    ch := make(dm, 3)
    
    fr fr Send values without blocking
    ch <- 1
    ch <- 2
    ch <- 3
    fr fr Channel is now full
    
    fr fr This would block until someone receives
    stan slay() {
        vibez.spill("Trying to send 4...")
        ch <- 4  fr fr This blocks until there's room
        vibez.spill("Sent 4!")
    }()
    
    fr fr Receive a value to make room
    timez.Sleep(1 * timez.Second)
    value := <-ch
    vibez.spill("Received:", value)  fr fr Should be 1
    
    fr fr Now the goroutine should be able to send
    timez.Sleep(1 * timez.Second)
}
Closing Channels
slay main() {
    fr fr Create a channel
    jobs := make(dm, 5)
    done := make(dm)
    
    fr fr Start a worker goroutine
    stan slay() {
        fr fr Process jobs until channel is closed
        bestie {
            job, more := <-jobs
            lowkey more {
                vibez.spill("Received job", job)
            } highkey {
                vibez.spill("Channel closed, stopping worker")
                done <- based
                yolo
            }
        }
    }()
    
    fr fr Send jobs
    bestie j := 1; j <= 3; j++ {
        jobs <- j
    }
    
    fr fr Close the channel to signal no more jobs
    close(jobs)
    
    fr fr Wait for worker to finish
    <-done
}

Range Over Channels

The flex keyword makes iterating over channel values a whole vibe:

slay main() {
    fr fr Create a channel
    ch := make(dm)
    
    fr fr Start a sender goroutine
    stan slay() {
        fr fr Send some messages
        words := []tea{"purr", "slay", "bestie", "no cap"}
        bestie _, word := flex words {
            ch <- word
        }
        
        fr fr Close channel when done sending
        close(ch)
    }()
    
    fr fr Range over the channel until it's closed
    bestie word := flex ch {
        vibez.spill("Received:", word)
    }
    
    vibez.spill("Channel closed, loop exited")
}

Select Statement

The select statement is the ultimate multitasker, letting you work with multiple channels at once:

slay main() {
    fr fr Create channels
    ch1 := make(dm)
    ch2 := make(dm)
    quit := make(dm)
    
    fr fr Start goroutines that send messages
    stan slay() {
        timez.Sleep(1 * timez.Second)
        ch1 <- "message from channel 1"
    }()
    
    stan slay() {
        timez.Sleep(2 * timez.Second)
        ch2 <- "message from channel 2"
    }()
    
    stan slay() {
        timez.Sleep(3 * timez.Second)
        quit <- 0
    }()
    
    fr fr Process messages using select
    bestie {
        select {
        mood msg := <-ch1:
            vibez.spill("Received from ch1:", msg)
        mood msg := <-ch2:
            vibez.spill("Received from ch2:", msg)
        mood <-quit:
            vibez.spill("Quit signal received, exiting...")
            ghosted
        mood <-timez.After(500 * timez.Millisecond):
            vibez.spill("Timeout, no message received in 500ms")
        }
    }
}

Channel Best Practices

  • Use buffered channels when you know the exact number of messages
  • Use unbuffered channels when you need guaranteed delivery
  • Close channels from the sender side, never from the receiver
  • Check if a channel is closed when receiving with val, ok := <-ch
  • Use select with a default case for non-blocking operations
  • Use select with timez.After for timeouts
