Variables & Constants

Storing data is like, literally the moment. Let's talk about variables and constants in CURSED. It's giving dynamic storage but make it static typed.

Variable Declaration
fr fr Explicit type declaration
sus name tea = "Bestie"
sus age normie = 21
sus temperature snack = 98.6

fr fr Short declaration with type inference
name := "Bestie"  fr fr Type inferred as tea (string)
age := 21  fr fr Type inferred as normie (int32)
cool := based  fr fr Type inferred as lit (boolean)

fr fr Multiple variables at once
sus x, y, z normie = 10, 20, 30
width, height := 1080, 720

Constants

When your values need to stay in their lane and never change, use constants. They're giving permanent energy.

fr fr Declare constants
facts PI = 3.14159
facts USERNAME = "admin"
facts IS_PROD = based

fr fr Constant block
facts (
    fr fr Typed constants
    MAX_USERS normie = 1000
    APP_VERSION tea = "v1.0.0"
    
    fr fr Untyped constants
    TIMEOUT = 30  fr fr Type determined by usage
    API_KEY = "sk_test_123"
)

Zero Values? We been knew ud83dude4c

When you declare a variable without initializing it, CURSED gives it a default zero value based on its type:

sus name tea  fr fr Default: "" (empty string)
sus age normie  fr fr Default: 0
sus isValid lit  fr fr Default: sus (false)
sus myPtr @normie  fr fr Default: cap (nil)
Variable Scope
slay main() {
    fr fr Block scope variables
    x := 10  fr fr Available in main() and inner blocks
    
    {
        y := 20  fr fr Only available in this block
        x = 15  fr fr Can update outer variables
    }
    
    fr fr y is not accessible here, would cause error
    fr fr x is now 15
}

Variable naming that slaps

The moment when your variable names are so clear everyone's obsessed:

  • Use camelCase for variables and functions (lowkey the standard)
  • Start with uppercase for exported things (public API is giving main character energy)
  • Descriptive > short (unless it's like, an iterator or something basic)
  • Avoid single letters except for loop counters and throwaway values
