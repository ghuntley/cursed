## 💡 Getting the Most Out of Discord

### Best Practices

**🎯 Ask Great Questions**
```
❌ Bad: "My code doesn't work, help!"

✅ Good: "I'm getting a 'channel closed' error when trying to 
send to a channel. Here's my code: [paste code]. I expected 
it to send the message, but it panics instead. Using CURSED 1.0."
```

**📝 Share Code Properly**
```cursed
# Use code blocks for better formatting
```cursed
yeet "vibez"
vibez.spill("This is properly formatted!")
```
# Or use snippets for longer code
```

**🤝 Help Others**
- Answer questions when you can
- Share resources and links
- Provide constructive feedback
- Celebrate others' achievements

## 🛡️ Community Guidelines

### Code of Conduct

**Be Respectful**
- Treat everyone with kindness and respect
- No harassment, discrimination, or hate speech
- Respect different skill levels and backgrounds

**Stay On Topic**
- Use appropriate channels for discussions
- Keep conversations relevant to CURSED
- Take lengthy off-topic discussions to DMs

**Help Create a Positive Environment**
- Encourage learning and experimentation
- Provide constructive feedback
- Share knowledge and resources freely
- Celebrate community achievements

### Reporting Issues

If you experience harassment or see inappropriate behavior:
1. **Report to moderators** - Use `!report` command or DM a moderator
2. **Block the user** - Right-click → Block if needed
3. **Document evidence** - Screenshots can help with investigations

Moderators are identified by special roles and are online during most hours.

## 📞 Getting Support

### Quick Help Channels
- **#beginners** - New to CURSED? Start here!
- **#installation-help** - Setup and configuration issues
- **#help-general** - General programming questions
- **#help-advanced** - Complex technical issues

### Response Time Expectations
- **Simple questions**: Usually answered within 1-2 hours
- **Complex issues**: May take several hours or a day
- **Bug reports**: Acknowledged within 24 hours
- **Feature requests**: Discussed within a week

### Before Asking for Help
1. **Search previous messages** - Your question might already be answered
2. **Check documentation** - Many answers are in the docs
3. **Try debugging** - Include what you've already tried
4. **Provide context** - Share relevant code and error messages
