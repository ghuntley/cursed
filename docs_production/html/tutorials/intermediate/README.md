# Intermediate Tutorial Series

Welcome to the CURSED intermediate tutorial series! This series builds upon the fundamentals and introduces more advanced language features.

## Prerequisites

Before starting this series, you should have completed the [Beginner Tutorial Series](../beginner/README.md) and be comfortable with:
- Basic CURSED syntax and keywords
- Variables, functions, and control flow
- Data structures (arrays, slices, structs, maps)
- Basic error handling

## What You'll Learn

This series covers advanced CURSED features that will make you a more effective developer:

### 🔧 Advanced Language Features
- Module system and package management
- Generic programming with type parameters
- Interface design and implementation
- Concurrency with goroutines and channels
- Pattern matching with complex scenarios
- Memory management and optimization

### 🚀 Development Skills
- Code organization and architecture
- Performance optimization techniques
- Testing and debugging strategies
- Best practices for maintainable code

## Learning Path

### For Experienced Developers
If you're coming from other languages like Go, Rust, or Java:
- Start with [Modules and Imports](01-modules-imports.md) to understand CURSED's unique approach
- Focus on [Generics](02-generics.md) and [Interfaces](03-interfaces.md) for type system features
- Dive into [Concurrency](04-concurrency.md) for parallel programming

### For Performance-Focused Development
If you're building high-performance applications:
- Complete [Generics](02-generics.md) for zero-cost abstractions
- Master [Concurrency](04-concurrency.md) for parallel execution
- Study [Memory Management](06-memory-management.md) for optimization

### For System Programming
If you're interested in systems development:
- Focus on [Memory Management](06-memory-management.md)
- Master [Concurrency](04-concurrency.md) primitives
- Learn [Advanced Pattern Matching](05-pattern-matching.md) for complex logic
