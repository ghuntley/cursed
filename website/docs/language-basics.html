data types

cursed has a rich type system that keeps your code type-safe while staying iconic:

basic types

primitive types

tea UTF-8 string
normie 32-bit signed integer
big_normie 64-bit signed integer
decimal 64-bit floating point
vibes boolean (based/sus)
cap null/nil value 
fr fr basic type examples
sus name tea = "bestie"
sus age normie = 22
sus balance big_normie = 1000000
sus height decimal = 5.6
sus is_icon vibes = based
sus nothing = cap

type inference

cursed can figure out types automatically, so you don't always need to specify them:

fr fr let cursed figure it out
sus auto_string = "cursed rocks!"  fr fr inferred as tea
sus auto_int = 42            fr fr inferred as normie
sus auto_float = 3.14         fr fr inferred as decimal
sus auto_bool = based        fr fr inferred as vibes

collections

cursed provides powerful collection types for organizing your data:

arrays and slices

fr fr arrays with fixed size
sus numbers = [5]normie{1, 2, 3, 4, 5}

fr fr slices are more flexible
sus names = []tea{"alice", "bob", "charlie"}
sus scores = []normie{95, 87, 92}

fr fr accessing elements
sus first_name = names[0]      fr fr "alice"
sus last_score = scores[2]     fr fr 92

fr fr slice operations
sus subset = names[1:3]         fr fr ["bob", "charlie"]
sus length = len(names)        fr fr 3

maps (dictionaries)

fr fr maps for key-value pairs
sus ages = tea[normie]{
    "alice": 25,
    "bob": 30,
    "charlie": 22,
}

fr fr accessing map values
sus alice_age = ages["alice"]    fr fr 25

fr fr checking if key exists
sus age, exists = ages["david"]
lowkey exists {
    io.printf("david is {} years old\n", age)
} highkey {
    io.println("david not found bestie")
}

fr fr adding/updating values
ages["eve"] = 28
delete(ages, "bob")

type conversions

convert between types safely and efficiently:

yeet "stdlib::strconv"

fr fr string to number conversions
sus num_str = "42"
sus float_str = "3.14159"

sus num, err = strconv.parse_int(num_str)
lowkey err == cap {
    io.printf("parsed number: {}\n", num)
}

sus pi, err2 = strconv.parse_float(float_str)
lowkey err2 == cap {
    io.printf("parsed float: {}\n", pi)
}

fr fr number to string conversions
sus age = 25
sus height = 5.8

sus age_str = strconv.int_to_string(age)
sus height_str = strconv.float_to_string(height)

fr fr type casting between numbers
sus big_num big_normie = 1000000
sus small_num = normie(big_num)  fr fr explicit cast
sus as_float = decimal(small_num)   fr fr 1000000.0

💡 type conversion tips

  • always check for errors when parsing strings to numbers
  • use explicit casts when converting between numeric types
  • be aware of potential precision loss when casting
  • cursed is strict about types - no implicit conversions!

scope & visibility

understand how cursed manages variable scope and function visibility:

variable scope

fr fr global scope
sus global_var = "accessible everywhere"

slay scope_demo() {
    fr fr function scope
    sus function_var = "only in this function"
    
    lowkey based {
        fr fr block scope
        sus block_var = "only in this block"
        io.println(global_var)    fr fr works
        io.println(function_var) fr fr works
        io.println(block_var)    fr fr works
    }
    
    fr fr block_var is not accessible here
    io.println(function_var) fr fr still works
}

function visibility

fr fr private function (lowercase first letter)
slay private_helper() {
    io.println("only accessible within this package")
}

fr fr public function (uppercase first letter)
slay Public_api() {
    io.println("accessible from other packages")
}

fr fr constants visibility
facts private_const = "package private"
facts PUBLIC_CONST = "exported constant"

🎯 visibility rules

  • lowercase first letter = private (package scope)
  • uppercase first letter = public (exported)
  • applies to functions, constants, types, and variables
  • no explicit public/private keywords needed
